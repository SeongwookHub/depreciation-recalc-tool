# 고정자산 감가상각비 재계산 검증 도구 (depreciation-audit-tool)

[![Tests](https://github.com/SeongwookHub/depreciation-audit-tool/actions/workflows/tests.yml/badge.svg)](https://github.com/SeongwookHub/depreciation-audit-tool/actions/workflows/tests.yml)

회사가 장부에 반영한 고정자산 감가상각비가 정액법·정률법·생산량비례법 계산 원리에 맞게 정확히 계산됐는지, 엑셀 고정자산대장 하나로 자동 재계산하고 차이 나는 자산만 짚어주는 파이썬 도구입니다.

## 1. 왜 만들었나

파트타임으로 회계법인 감사에 참여하면서 고정자산 계정을 담당한 적이 있습니다. 이 계정에서 가장 손이 많이 가는 작업 중 하나가 "회사가 장부에 반영한 당기 감가상각비가 맞는지" 검증하는 일이었습니다.

- 자산마다 취득일이 달라서 정액법이라도 월할상각(기중에 취득한 달수만큼만 상각)을 일일이 계산해야 하고,
- 정률법 자산은 취득 이후 연도별로 장부가액이 어떻게 줄어들었는지 처음부터 다시 굴려봐야 하고,
- 여기에 자산 처분이나 내용연수 재추정 같은 이벤트까지 섞이면 검증해야 할 경우의 수가 급격히 늘어납니다.

이걸 자산 수십~수백 건에 대해 엑셀 수식과 손 계산으로 대조하다 보면 시간이 오래 걸리고, 사람이 하는 일이라 실수도 나옵니다. "고정자산대장 하나만 넣으면 자동으로 재계산해서 틀린 자산만 보여주는 도구가 있으면 좋겠다"는 생각에서 이 프로젝트를 시작했습니다.

## 2. 주요 기능

- **정액법 재계산**: (취득원가 − 잔존가치) ÷ 내용연수 로 연간 상각액을 구하고, 당기 중 실제로 상각 대상이었던 개월수만큼 비례해서 계산합니다.
- **정률법 재계산**: 국세청 고시 내용연수별 상각률표를 적용해, 취득연도부터 당기까지 매년 장부가액(미상각잔액)을 순서대로 줄여가며 당기분을 계산합니다.
- **생산량비례법 재계산**: (취득원가 − 잔존가치) × (당기실제생산량 ÷ 총예정생산량)으로 당기 상각비를 계산합니다. `총예정생산량`/`당기실제생산량` 컬럼은 선택 항목이라, 파일에 없으면 해당 자산이 없는 것으로 보고 정상 동작합니다. (v1 한계: 취득 이후 누적생산량 이력은 추적하지 않고 당기 값만 반영합니다.)
- **월할상각 반영**: 연도 중간에 취득한 자산은 취득한 달부터 개월수를 세어 그 비율만큼만 당기 상각비로 인정합니다.
- **처분(매각)자산 처리**: 당기 중 자산을 처분한 경우, 처분한 달까지만 상각하고 그 이후는 계산에서 자동으로 제외합니다.
- **내용연수 재추정 처리**: 상각 도중 내용연수가 재조정된 자산은, 재추정 시점의 장부가액을 새로운 기준가액으로 삼아 남은 기간을 새 내용연수로 다시 상각합니다.
- **내용연수 종료 자산 처리**: 이미 상각이 끝난 자산은 당기 상각비를 0원으로 처리하고 더 이상 상각하지 않습니다.
- **컬럼명 유연 매핑**: 회사마다 고정자산대장의 컬럼명이 제각각이어도(예: "취득원가" 대신 "취득가액"), 코드 상단의 매핑 표 한 곳만 고치면 나머지 로직은 그대로 사용할 수 있습니다.
- **입력값 검증(데이터오류 시트)**: 내용연수<1, 취득원가≤0, 잔존가치≥취득원가처럼 계산이 애초에 불가능한 값을 가진 자산은 재계산을 시도하지 않고 크래시 없이 `데이터오류` 시트로 격리합니다. 자산마다 구체적인 오류 사유를 함께 표시해, 다른 정상 자산의 계산에는 영향을 주지 않으면서 원본 데이터의 어디를 고쳐야 하는지 바로 알 수 있습니다.
- **차이 자산만 별도 정리**: 재계산 금액과 회사가 장부에 반영한 금액을 비교해서, 차이가 있는 자산만 별도 시트로 뽑아 보여줍니다.
- **중요성 기준(Materiality) 자동 판정**: 회계 감사에서는 아무리 사소한 금액이라도 다 따지지 않고, "이 정도 금액 차이는 실제로 재무제표에 영향을 줄 만하다"고 볼 수 있는 기준선을 정해 놓고 그 이상인 것만 집중적으로 봅니다. 이 도구도 재계산 금액과 회사 반영 금액의 차이가 기준 금액(기본값 100만원, `MATERIALITY_THRESHOLD` 값으로 조정 가능) 이상이면 **"유의한 차이"**, 미만이면 **"경미한 차이"**로 자동 분류해서, 수십~수백 건의 불일치 중 어떤 자산부터 우선적으로 확인해야 하는지 알려줍니다.
- **자산군별 통계**: `재계산결과`의 `자산분류` 컬럼을 기준으로 자산분류별 전체건수·정상계산건수·불일치건수·불일치율·유의한차이건수·오류건수를 집계해 `자산군별통계` 시트로 별도 정리하고, 콘솔에도 요약을 출력합니다.
- **규칙 기반 + AI 하이브리드 원인 분류**: "유의한 차이"로 분류된 자산에 대해, 처분일 미반영·내용연수 재추정 미반영·내용연수 종료 자산 지속상각처럼 이미 계산된 정보만으로 원인이 명확한 케이스는 먼저 규칙으로 분류합니다. 규칙으로 설명되지 않는 애매한 경우에만 Claude API를 호출해 원인을 추정합니다(아래 항목 참고). 결과는 `추정원인`(코멘트)과 `추정원인출처`(`규칙`/`AI`/`-`) 두 컬럼에 담깁니다.
- **감사자 검토란**: `차이자산`/`유의한차이자산` 시트에 감사자가 직접 채워 넣을 수 있는 `검토자`/`검토의견`/`검토일` 빈 컬럼을 추가해 두었습니다(엑셀에서 연두색으로 구분 표시).
- **다기간(연도별) 비교**: `COMPARISON_YEARS`에 여러 회계연도를 지정하면(예: `[2023, 2024, 2025]`), 자산별 연도별 재계산 상각비를 `연도별추이`(long format) / `연도별요약`(연도를 컬럼으로 펼친 피벗) 두 시트로 정리하고, 전년대비 `YOY_ANOMALY_THRESHOLD_PCT`(기본 20%) 이상 증감한 자산을 "경고"로, 전년도 0원에서 새로 발생한 상각비는 "신규발생"으로 자동 표시합니다. 기본값인 `[FY_YEAR]`(1개 연도)만 두면 이 시트들은 생성되지 않고 기존과 동일하게 단일 연도만 계산합니다.
- **자동 테스트(pytest) 52개로 계산 로직 검증**: "이 프로그램의 계산이 정말 맞게 짜여 있는가?"를 사람이 매번 눈으로 확인하는 대신, 정액법·정률법·생산량비례법·월할상각·처분자산·내용연수재추정·중요성판정·자산군별통계·규칙기반 원인분류·다기간비교 같은 핵심 계산 규칙 하나하나에 대해 "이런 상황을 넣으면 이런 결과가 나와야 정상"이라는 정답을 미리 정해 둔 자동 검증 테스트 52개를 마련해 두었습니다. 코드를 수정한 뒤에도 이 테스트들을 실행해 보면 계산 로직이 여전히 올바른지 바로 확인할 수 있습니다.
- **GitHub Actions를 통한 자동 검증 + 통과 배지**: 코드를 GitHub 저장소에 올릴(push) 때마다 위 52개 테스트가 자동으로 실행됩니다. 사람이 직접 테스트를 돌려보지 않아도, 이 README 맨 위에 있는 **Tests 배지**로 지금 이 순간 테스트가 전부 통과했는지(초록색) 아니면 어딘가 깨졌는지(빨간색)를 바로 확인할 수 있습니다.
- **Claude API 기반 AI 추정원인(선택 기능)**: 위 규칙 기반 분류로 설명되지 않는 "유의한 차이" 자산에 대해서는, 차이금액·경과개월수(취득 후 몇 개월이 지났는지)·상각방법·내용연수 재추정 여부·처분 여부 같은 정보를 앤트로픽의 AI 모델(Claude)에게 전달해서 "왜 이런 차이가 발생했을 가능성이 높은지"를 한 줄짜리 코멘트로 추정받습니다. 이 기능은 선택 사항이라 `.env` 파일에 `ANTHROPIC_API_KEY`가 없거나 API 호출 중 오류가 나더라도 `추정원인` 컬럼만 "-"로 표시될 뿐, 나머지 재계산 로직은 평소와 똑같이 정상 동작합니다.

## 3. 실행 방법

### 필요한 프로그램

- Python 3.9 이상
- 라이브러리: `pandas`, `openpyxl`, `anthropic`, `python-dotenv`

### 설치

터미널(PowerShell)에서 아래 명령어를 실행해 필요한 라이브러리를 설치합니다.

```powershell
python -m pip install pandas openpyxl anthropic python-dotenv
```

AI 추정원인 기능을 쓰려면 프로젝트 폴더의 `.env` 파일에 다음과 같이 API 키를 넣습니다(없어도 나머지 기능은 정상 동작합니다).

```
ANTHROPIC_API_KEY=여기에_실제_키_입력
```

### 실행

이 프로젝트 폴더에서 아래 명령어를 실행합니다.

```powershell
python recalc.py
```

실행하면 먼저 어떤 컬럼을 어떻게 인식했는지 안내 메시지가 출력되고, 이어서 `recalc_result.xlsx` 파일이 같은 폴더에 생성됩니다.

### 테스트 실행

계산 로직이 올바르게 동작하는지 pytest로 검증할 수 있습니다.

```powershell
python -m pip install pytest
pytest test_recalc.py -v
```

### 다른 회사 파일에 적용하려면

1. `recalc.py` 상단의 `IN_PATH`를 검증하려는 고정자산대장 엑셀 파일 경로로 바꿉니다.
2. `COLUMN_MAP` 딕셔너리에서 왼쪽(자산명, 취득원가 등 항목 이름)은 그대로 두고, 오른쪽 값만 실제 파일의 컬럼명으로 바꿉니다.
3. `python recalc.py`를 실행합니다.

### 여러 회계연도를 한 번에 비교하려면

`recalc.py` 상단의 `COMPARISON_YEARS`를 원하는 연도 목록으로 바꿉니다(기본값은 `[FY_YEAR]`, 즉 당기 1개년만 계산).

```python
COMPARISON_YEARS = [2023, 2024, 2025]
```

이렇게 두 개 이상의 연도를 지정하면 `연도별추이`/`연도별요약` 시트가 추가로 생성되고, 전년대비 증감이 `YOY_ANOMALY_THRESHOLD_PCT`(기본 20%) 이상이면 자동으로 "경고" 표시됩니다.

## 4. 사용 예시

이 저장소에는 실제 데이터 대신 `sample_asset_ledger.xlsx`라는 가상의 고정자산대장 샘플이 들어 있습니다. 정액법 자산과 정률법 자산이 섞여 있고, 취득 시점이 서로 다른 여러 해에 걸쳐 있으며, 처분자산·내용연수 재추정 자산·내용연수 종료 자산 같은 특수 케이스도 포함되어 있습니다. 회사가 반영한 감가상각비 중 일부는 실제로 계산되어야 할 금액과 다르게(의도적으로 오류가 나도록) 넣어 두었습니다.

이 샘플 파일로 `recalc.py`를 실행하면 다음과 같은 결과를 얻습니다.

- 전체 13건의 자산 중 10건은 회사 반영 금액과 재계산 금액이 정확히 일치합니다.
- 3건은 재계산 금액과 회사 반영 금액이 다르게 나와 "불일치"로 표시됩니다. 예를 들어 한 정액법 자산은 회사가 12,500,000원을 반영했지만 재계산하면 12,000,000원이 맞는 값으로 나오고, 한 정률법 자산은 회사가 12,500,000원을 반영했지만 재계산하면 13,180,933원이 맞는 값으로 나오는 식입니다.
- 당기 중 처분된 자산은 처분한 달까지만 6개월치가 정상적으로 계산되고, 내용연수가 재추정된 자산은 재추정 이후 새 기준으로 계산되며, 이미 내용연수가 끝난 자산은 당기 상각비 0원으로 정확히 처리됩니다.
- 결과 파일에는 기본적으로 5개의 시트가 생깁니다. `재계산결과` 시트는 13건 전체의 상세 계산 내역을, `차이자산`/`유의한차이자산` 시트는 불일치 자산을 중요성 기준별로, `데이터오류` 시트는 계산 자체가 불가능한 자산을, `자산군별통계` 시트는 자산분류별 집계를 보여줍니다. `COMPARISON_YEARS`에 2개 이상의 연도를 지정하면 `연도별추이`/`연도별요약` 시트가 추가됩니다.

### 컬럼명이 다른 회사 파일 예시 (sample_asset_ledger_v2.xlsx)

`sample_asset_ledger_v2.xlsx`는 컬럼명이 기본 `COLUMN_MAP`과 다른(예: "취득원가" 대신 "취득 원가", "잔존가치" 대신 "잔존가액", "내용연수(년)" 대신 "내용년수(년)") 회사 파일을 흉내 낸 샘플입니다. "3. 실행 방법 → 다른 회사 파일에 적용하려면"의 안내대로 `COLUMN_MAP`을 이 파일의 실제 컬럼명에 맞게 바꾼 뒤 실행하면 됩니다.

이 샘플(27건)로 실행한 결과는 `recalc_result_v2.xlsx`에서 확인할 수 있습니다.

- 27건 중 24건은 계산 가능한 값이라 정상적으로 재계산되고, 3건은 애초에 계산이 불가능한 값이라 재계산을 시도하지 않고 `데이터오류` 시트로 격리됩니다. 예를 들어 내용연수가 0년이거나 음수(-5년)인 자산, 잔존가치가 취득원가보다 큰 자산이 여기 해당합니다.
- 정상 계산된 24건 중 8건은 회사 반영 금액과 재계산 금액이 달라 `차이자산` 시트에 표시되고, 그중 5건은 중요성 기준(100만원) 이상 차이가 나 `유의한차이자산` 시트에 별도로 정리됩니다.
- `자산군별통계` 시트에는 유형자산/무형자산 분류별로 전체건수·정상계산건수·불일치건수·불일치율·유의한차이건수·오류건수가 집계되어 나옵니다.

## 5. 향후 개선 방향

- 국세청 상각률표를 더 넓은 내용연수 범위까지 정확한 공시 수치로 채워 넣기
- 자본적 지출(추가 취득원가 반영)이 있었던 자산 처리 지원
- 월할이 아닌 일할상각 옵션 지원
- 생산량비례법에 취득 이후 누적생산량 이력을 반영해 다년간 상각누계액을 정확히 추적
- 웹 화면이나 간단한 UI를 붙여서 파이썬을 몰라도 파일만 올리면 결과를 받을 수 있도록 개선
- 전기 대비 롤포워드(Roll-forward) 검증 — 기초잔액 + 당기증감 = 기말잔액 정합성 확인
- 전체중요성/수행중요성/AMPT(허용가능한 오류금액) 3단계 중요성 기준 적용 — 현재의 단일 임계치(`MATERIALITY_THRESHOLD`) 대신 감사 실무의 3단계 기준 반영
- 자산별 변경이력(상각중단, 내용연수·상각방법 변경) 반영 — 현재는 내용연수 재추정만 처리하므로, 상각중단이나 상각방법 변경 이력까지 확장

## 6. 주의사항

이 저장소에 포함된 `sample_asset_ledger.xlsx`는 검증 로직을 설명하고 테스트하기 위해 처음부터 새로 만든 **가상의 샘플 데이터**입니다. 실제 회사명, 실제 자산 정보, 실제 감사 대상 데이터는 전혀 포함되어 있지 않습니다.
