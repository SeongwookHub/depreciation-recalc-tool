# 고정자산 감가상각비 재계산 검증 도구 (depreciation-audit-tool)

[![Tests](https://github.com/SeongwookHub/depreciation-audit-tool/actions/workflows/tests.yml/badge.svg)](https://github.com/SeongwookHub/depreciation-audit-tool/actions/workflows/tests.yml)

회사가 장부에 반영한 고정자산 감가상각비가 정액법·정률법·이중체감법·연수합계법·생산량비례법 계산 원리에 맞게 정확히 계산됐는지, 엑셀 고정자산대장 하나로 자동 재계산하고 차이 나는 자산만 짚어주는 파이썬 도구입니다.

## 1. 왜 만들었나

파트타임으로 회계법인 감사에 참여하면서 고정자산 계정을 담당한 적이 있습니다. 이 계정에서 가장 손이 많이 가는 작업 중 하나가 "회사가 장부에 반영한 당기 감가상각비가 맞는지" 검증하는 일이었습니다.

- 자산마다 취득일이 달라서 정액법이라도 월할상각(기중에 취득한 달수만큼만 상각)을 일일이 계산해야 하고,
- 정률법 자산은 취득 이후 연도별로 장부가액이 어떻게 줄어들었는지 처음부터 다시 굴려봐야 하고,
- 여기에 자산 처분이나 내용연수 재추정 같은 이벤트까지 섞이면 검증해야 할 경우의 수가 급격히 늘어납니다.

이걸 자산 수십~수백 건에 대해 엑셀 수식과 손 계산으로 대조하다 보면 시간이 오래 걸리고, 사람이 하는 일이라 실수도 나옵니다. "고정자산대장 하나만 넣으면 자동으로 재계산해서 틀린 자산만 보여주는 도구가 있으면 좋겠다"는 생각에서 이 프로젝트를 시작했습니다.

## 2. 주요 기능

- **정액법 재계산**: (취득원가 − 잔존가치) ÷ 내용연수 로 연간 상각액을 구하고, 당기 중 실제로 상각 대상이었던 개월수만큼 비례해서 계산합니다.
- **정률법 재계산**: 국세청 고시 내용연수별 상각률표를 적용해, 취득연도부터 당기까지 매년 장부가액(미상각잔액)을 순서대로 줄여가며 당기분을 계산합니다. 미상각잔액이 취득가액의 5% 이하로 떨어지는 과세기간에는 법인세법 시행규칙 [별표4] 특례에 따라 잔여 전액을 그 해에 상각합니다.
- **이중체감법 재계산**: 상각률(2 ÷ 내용연수)을 매년 장부가액에 곱해가며 계산합니다(정률법과 계산 구조는 같지만 상각률표가 아니라 정액법 상각률의 2배를 직접 쓰고, 세법상 5% 특례 대신 잔존가치를 하한으로만 씁니다).
- **연수합계법 재계산**: 내용연수 합계(1+2+…+N)를 분모로, 잔여내용연수를 분자로 한 그 해의 몫을 상각대상금액에 곱해 계산합니다(1년차에 가장 많이, 마지막 해에 가장 적게 상각).
- **생산량비례법 재계산**: (취득원가 − 잔존가치) × (당기실제생산량 ÷ 총예정생산량)으로 당기 상각비를 계산합니다. `총예정생산량`/`당기실제생산량` 컬럼은 선택 항목이라, 파일에 없으면 해당 자산이 없는 것으로 보고 정상 동작합니다. `전기말누적생산량`까지 함께 입력하면 전기말/당기말 감가상각누계액도 수식으로 연결됩니다.
- **엑셀 수식으로 연결된 결과**: 상각률·감가상각대상금액·재계산 감가상각비·전기말/당기말 감가상각누계액·경과개월수·당기해당월수 같은 계산 결과는 고정된 값이 아니라 실제 엑셀 수식(`MONTH()`/`YEAR()`/`VLOOKUP` 등)으로 셀에 연결되어 있어, 셀을 눌러 어떻게 계산됐는지 확인하거나 입력값을 바꿔가며 재계산 결과가 어떻게 달라지는지 직접 시험해볼 수 있습니다(당기 중 자본적지출·감가상각중단처럼 한 셀 수식으로 표현하기엔 지나치게 복잡해지는 극소수 케이스만 계산값을 그대로 둡니다).
- **기준정보 안내 시트**: 결과 파일 맨 앞 탭에 이 프로그램이 어떻게 동작하는지, 결과 파일이 어떤 과정을 거쳐 만들어지는지, 각 시트가 무엇을 보여주는지를 설명하는 안내문과 함께, 기준일·중요성(AMPT/중요성/수행중요성) 같은 이번 실행의 기준값을 모아 둡니다.
- **월할상각 반영**: 연도 중간에 취득한 자산은 취득한 달부터 개월수를 세어 그 비율만큼만 당기 상각비로 인정합니다.
- **처분(매각)자산 처리**: 당기 중 자산을 처분한 경우, 처분한 달까지만 상각하고 그 이후는 계산에서 자동으로 제외합니다.
- **내용연수 재추정 처리**: 상각 도중 내용연수가 재조정된 자산은, 재추정 시점의 장부가액을 새로운 기준가액으로 삼아 남은 기간을 새 내용연수로 다시 상각합니다.
- **내용연수 종료 자산 처리**: 이미 상각이 끝난 자산은 당기 상각비를 0원으로 처리하고 더 이상 상각하지 않습니다.
- **컬럼명 유연 매핑(동의어 자동 인식 + AI 추천)**: 회사마다 고정자산대장의 컬럼명이 제각각이어도(예: "취득원가" 대신 "취득가액", "자산명" 대신 "자산코드") 미리 등록된 동의어 사전으로 코드를 고치지 않고도 자동으로 매칭을 시도합니다. 그래도 못 찾으면 (API 키가 있으면 Claude로, 없으면 문자열 유사도로) "혹시 이 컬럼 아닌가요?" 후보를 보여주되, 실제 매핑은 절대 자동으로 반영하지 않고 사람이 `COLUMN_MAP`을 보고 확정하도록 합니다(컬럼을 잘못 매칭하면 코멘트 하나가 아니라 전체 재계산 결과가 틀어지기 때문). 상각방법 값도 "정액"/"정률"/"SL"/"DDB" 같은 줄임말·영문 약자를 자동으로 정규화하고, 그래도 인식 안 되는 값은 전체 실행을 막지 않고 그 자산만 `데이터오류` 시트로 격리합니다.
- **입력값 검증(데이터오류 시트)**: 내용연수<1, 취득원가≤0, 잔존가치≥취득원가처럼 계산이 애초에 불가능한 값을 가진 자산은 재계산을 시도하지 않고 크래시 없이 `데이터오류` 시트로 격리합니다. 자산마다 구체적인 오류 사유를 함께 표시해, 다른 정상 자산의 계산에는 영향을 주지 않으면서 원본 데이터의 어디를 고쳐야 하는지 바로 알 수 있습니다.
- **차이 자산만 별도 정리**: 재계산 금액과 회사가 장부에 반영한 금액을 비교해서, 차이가 있는 자산만 별도 시트로 뽑아 보여줍니다.
- **중요성 기준(Materiality) 자동 판정**: 회계 감사에서는 아무리 사소한 금액이라도 다 따지지 않고, "이 정도 금액 차이는 실제로 재무제표에 영향을 줄 만하다"고 볼 수 있는 기준선을 정해 놓고 그 이상인 것만 집중적으로 봅니다. 이 도구도 재계산 금액과 회사 반영 금액의 차이가 AMPT(기본값 100만원, `MATERIALITY_THRESHOLD` 값으로 조정 가능) 이상이면 **"유의한 차이"**, 미만이면 **"경미한 차이"**로 자동 분류해서, 수십~수백 건의 불일치 중 어떤 자산부터 우선적으로 확인해야 하는지 알려줍니다. 기준정보 시트에는 이 AMPT와 함께 전체 재무제표 수준 중요성·수행중요성(PM)도 참고용으로 표시됩니다(둘은 판정 로직 자체에는 관여하지 않습니다).
- **자산군별 통계**: `재계산결과`의 `자산분류` 컬럼을 기준으로 자산분류별 전체건수·정상계산건수·불일치건수·불일치율·유의한차이건수·오류건수를 집계해 `자산군별통계` 시트로 별도 정리하고, 콘솔에도 요약을 출력합니다.
- **규칙 기반 + AI 하이브리드 원인 분류**: "유의한 차이"로 분류된 자산에 대해, 처분일 미반영·내용연수 재추정 미반영·내용연수 종료 자산 지속상각처럼 이미 계산된 정보만으로 원인이 명확한 케이스는 먼저 규칙으로 분류합니다. 규칙으로 설명되지 않는 애매한 경우에만 Claude API를 호출해 원인을 추정합니다(아래 항목 참고). 결과는 `추정원인`(코멘트)과 `추정원인출처`(`규칙`/`AI`/`-`) 두 컬럼에 담깁니다.
- **감사자 검토란**: `차이자산`/`유의한차이자산` 시트에 감사자가 직접 채워 넣을 수 있는 `검토자`/`검토의견`/`검토일` 빈 컬럼을 추가해 두었습니다(엑셀에서 연두색으로 구분 표시).
- **다기간(연도별) 비교**: `COMPARISON_YEARS`에 여러 회계연도를 지정하면(예: `[2023, 2024, 2025]`), 자산별 연도별 재계산 상각비를 `연도별추이`(long format) / `연도별요약`(연도를 컬럼으로 펼친 피벗) 두 시트로 정리하고, 전년대비 `YOY_ANOMALY_THRESHOLD_PCT`(기본 20%) 이상 증감한 자산을 "경고"로, 전년도 0원에서 새로 발생한 상각비는 "신규발생"으로 자동 표시합니다. 기본값인 `[FY_YEAR]`(1개 연도)만 두면 이 시트들은 생성되지 않고 기존과 동일하게 단일 연도만 계산합니다.
- **자동 테스트(pytest) 132개로 계산 로직 검증**: "이 프로그램의 계산이 정말 맞게 짜여 있는가?"를 사람이 매번 눈으로 확인하는 대신, 정액법·정률법·이중체감법·연수합계법·생산량비례법·월할상각·처분자산·내용연수재추정(방법변경 포함)·자본적지출·감가상각중단·중요성판정·자산군별통계·규칙기반 원인분류·다기간비교 같은 핵심 계산 규칙 하나하나에 대해 "이런 상황을 넣으면 이런 결과가 나와야 정상"이라는 정답을 미리 정해 둔 자동 검증 테스트를 마련해 두었습니다. 코드를 수정한 뒤에도 이 테스트들을 실행해 보면 계산 로직이 여전히 올바른지 바로 확인할 수 있습니다.
- **GitHub Actions를 통한 자동 검증 + 통과 배지**: 코드를 GitHub 저장소에 올릴(push) 때마다 위 테스트가 자동으로 실행됩니다. 사람이 직접 테스트를 돌려보지 않아도, 이 README 맨 위에 있는 **Tests 배지**로 지금 이 순간 테스트가 전부 통과했는지(초록색) 아니면 어딘가 깨졌는지(빨간색)를 바로 확인할 수 있습니다.
- **Claude API 기반 AI 추정원인(선택 기능)**: 위 규칙 기반 분류로 설명되지 않는 "유의한 차이" 자산에 대해서는, 차이금액·경과개월수(취득 후 몇 개월이 지났는지)·상각방법·내용연수 재추정 여부·처분 여부 같은 정보를 앤트로픽의 AI 모델(Claude)에게 전달해서 "왜 이런 차이가 발생했을 가능성이 높은지"를 한 줄짜리 코멘트로 추정받습니다. 이 기능은 선택 사항이라 `.env` 파일에 `ANTHROPIC_API_KEY`가 없거나 API 호출 중 오류가 나더라도 `추정원인` 컬럼만 "-"로 표시될 뿐, 나머지 재계산 로직은 평소와 똑같이 정상 동작합니다.

## 3. 실행 방법

### 필요한 프로그램

- Python 3.9 이상
- 라이브러리: `pandas`, `openpyxl`, `anthropic`, `python-dotenv`

### 설치

터미널(PowerShell)에서 아래 명령어를 실행해 필요한 라이브러리를 설치합니다.

```powershell
python -m pip install pandas openpyxl anthropic python-dotenv
```

AI 추정원인 기능을 쓰려면 프로젝트 폴더의 `.env` 파일에 다음과 같이 API 키를 넣습니다(없어도 나머지 기능은 정상 동작합니다).

```
ANTHROPIC_API_KEY=여기에_실제_키_입력
```

### 실행

이 프로젝트 폴더에서 아래 명령어를 실행합니다.

```powershell
python recalc.py
```

실행하면 먼저 어떤 컬럼을 어떻게 인식했는지 안내 메시지가 출력되고, 이어서 `recalc_result.xlsx` 파일이 같은 폴더에 생성됩니다.

### 테스트 실행

계산 로직이 올바르게 동작하는지 pytest로 검증할 수 있습니다.

```powershell
python -m pip install pytest
pytest test_recalc.py -v
```

### 다른 회사 파일에 적용하려면

회사마다 고정자산대장의 컬럼명(자산명 대신 자산코드, 취득원가 대신 취득가액 등)과 상각방법 표기(정액법 대신 "정액"만 쓰는 경우 등)가 제각각일 수 있습니다. 이 도구는 3단계로 이를 흡수합니다.

1. **정확 일치**: `COLUMN_MAP`에 지정된 이름이 파일에 그대로 있으면 사용합니다(기존 방식).
2. **동의어 자동 매칭**(신규): 정확히 일치하는 이름이 없으면, `recalc.py`의 `COLUMN_SYNONYMS`에 미리 등록된 흔한 별칭(예: 자산명 ↔ 자산코드/품목명, 취득원가 ↔ 취득가액/취득금액) 중 하나가 있는지 자동으로 찾아 씁니다. **`COLUMN_MAP`을 고치지 않아도 되는 경우가 많습니다** — 실행 시 콘솔의 "컬럼 인식 결과"에 `(동의어 매칭: '실제컬럼명')`으로 어떤 컬럼이 자동으로 매칭됐는지 항상 표시되니, 실행할 때마다 확인하는 습관을 들이는 게 좋습니다.
3. **실패 시 추천(자동 반영 안 됨)**: 필수 컬럼을 1·2단계로도 못 찾으면 지금처럼 실행이 중단되지만, `ANTHROPIC_API_KEY`가 설정돼 있으면 Claude가(없으면 문자열 유사도로) "혹시 이 컬럼 아닌가요?" 후보를 에러 메시지에 함께 보여줍니다. **이 추천은 참고용일 뿐 자동으로 반영되지 않습니다** — "장부가액"처럼 취득원가와 개념 자체가 다를 수 있는 컬럼은 사람이 직접 판단해서 `COLUMN_MAP`을 고쳐야 하기 때문입니다.

상각방법 값도 마찬가지로 "정액"/"정률"/"SL"/"DDB"처럼 흔한 줄임말·영문 약자는 `METHOD_ALIASES` 사전으로 자동 정규화됩니다. 사전에도 없는 완전히 낯선 값(오탈자 등)은 계산을 시도하지 않고 그 자산만 `데이터오류` 시트로 격리되며, 다른 자산의 계산에는 영향을 주지 않습니다(예전에는 전체 실행이 중단됐습니다).

그래도 컬럼을 못 찾거나 잘못 매칭됐다면, 아래처럼 `COLUMN_MAP`에서 해당 항목만 실제 파일의 컬럼명으로 직접 지정하면 됩니다(이 경우 1단계 정확 일치로 항상 우선 적용됩니다).

1. `recalc.py` 상단의 `IN_PATH`를 검증하려는 고정자산대장 엑셀 파일 경로로 바꿉니다.
2. 필요하다면 `COLUMN_MAP` 딕셔너리에서 왼쪽(자산명, 취득원가 등 항목 이름)은 그대로 두고, 오른쪽 값만 실제 파일의 컬럼명으로 바꿉니다.
3. `python recalc.py`를 실행합니다.

```python
COLUMN_MAP = {
    "자산명": "자산명",
    "취득원가": "취득 원가",       # <- 실제 파일의 컬럼명으로 교체
    "잔존가치": "잔존가액",        # <- 실제 파일의 컬럼명으로 교체
    "내용연수": "내용년수(년)",    # <- 실제 파일의 컬럼명으로 교체
    ...
}
```

### 여러 회계연도를 한 번에 비교하려면

`recalc.py` 상단의 `COMPARISON_YEARS`를 원하는 연도 목록으로 바꿉니다(기본값은 `[FY_YEAR]`, 즉 당기 1개년만 계산).

```python
COMPARISON_YEARS = [2023, 2024, 2025]
```

이렇게 두 개 이상의 연도를 지정하면 `연도별추이`/`연도별요약` 시트가 추가로 생성되고, 전년대비 증감이 `YOY_ANOMALY_THRESHOLD_PCT`(기본 20%) 이상이면 자동으로 "경고" 표시됩니다.

## 4. 사용 예시

이 저장소에는 실제 데이터 대신 `sample_asset_ledger.xlsx`라는 가상의 고정자산대장 샘플이 들어 있습니다(133건, `generate_sample.py`로 재현 가능). 정액법·정률법·이중체감법·연수합계법·생산량비례법 자산이 모두 섞여 있고, 취득 시점이 서로 다른 여러 해에 걸쳐 있으며, 처분·내용연수 재추정(상각방법 변경 포함)·자본적지출·감가상각중단과 이 이벤트들이 두세 개씩 겹치는 조합 케이스, 계산 자체가 불가능한 데이터오류 케이스까지 감가상각 재계산에서 실제로 마주칠 수 있는 경우의 수를 폭넓게 담고 있습니다. 회사가 반영한 감가상각비 중 일부는 실제로 계산되어야 할 금액과 다르게(의도적으로 오류가 나도록) 넣어 두었습니다.

이 샘플 파일로 `recalc.py`를 실행하면 다음과 같은 결과를 얻습니다(`recalc_result.xlsx`).

- 전체 133건 중 122건은 계산 가능한 값이라 정상적으로 재계산되고, 11건은 애초에 계산이 불가능한 값(내용연수 0년·음수, 취득원가 0 이하, 잔존가치≥취득원가 등)이라 재계산을 시도하지 않고 `데이터오류` 시트로 격리됩니다.
- 정상 계산된 122건 중 8건은 회사 반영 금액과 재계산 금액이 달라 `차이자산` 시트에 표시됩니다(전부 AMPT 100만원 미만의 "경미한 차이"). 예를 들어 `처분_연수합계_당기중_유형01`은 당기 중 처분을 반영했어야 하는데 회사가 다른 금액을 그대로 반영했고, `자본적지출_정률_당기중_01`은 당기 중 자본적지출이 있었는데 회사가 반영한 금액이 재계산 금액과 어긋나며, `방법변경_연수합계to정률_01`은 기중 상각방법 변경(연수합계법→정률법)이 반영되지 않아 차이가 납니다.
- 당기 중 처분된 자산은 처분한 달까지만 계산되고, 내용연수가 재추정된 자산은 재추정 이후 새 기준으로(상각방법이 바뀌는 경우 새 방법으로), 자본적지출이 있었던 자산은 지출액을 가산한 장부가액을 잔여 내용연수로, 감가상각중단이 있었던 자산은 중단기간을 0원 처리 후 그만큼 내용연수를 연장해 계산합니다.
- `재계산결과` 시트에는 회사반영 수치 외에도 경과개월수·당기해당월수·적용내용연수(개월)·상각률·감가상각대상금액·재계산 감가상각비·전기말/당기말 감가상각누계액 등 재계산 과정에서 도출되는 숫자들이 실제 엑셀 수식(`MONTH()`/`YEAR()`/`VLOOKUP` 등)으로 연결되어 있어, 입력값을 바꿔보면서 계산 과정을 직접 검토할 수 있습니다. 당기 중 자본적지출·감가상각중단이 있어 세그먼트 재구성 로직을 단일 수식으로 표현하기 어려운 극소수 케이스만 계산값을 그대로 둡니다.
- 헤더 색상으로 컬럼을 5개 그룹(취득값·이벤트·계산과정 / 당기 비교 / 전기말 누계 비교 / 당기말 누계 비교 / 분류·메타)으로 구분해 두어, 컬럼이 많아도 어느 값이 어느 성격인지 한눈에 구분할 수 있습니다.
- 결과 파일에는 `기준정보`/`재계산결과`/`차이자산`/`유의한차이자산`/`데이터오류`/`자산군별통계`와 상각률 수식의 근거가 되는 `상각률표(별표4)` 시트가 생깁니다. `COMPARISON_YEARS`에 2개 이상의 연도를 지정하면 `연도별추이`/`연도별요약` 시트가 추가됩니다.

## 5. 향후 개선 방향

- 월할이 아닌 일할상각 옵션 지원
- 웹 화면이나 간단한 UI를 붙여서 파이썬을 몰라도 파일만 올리면 결과를 받을 수 있도록 개선
- 전체중요성/수행중요성(PM)도 참고용 표시를 넘어 실제 3단계 판정 로직(명백히 사소한 차이/경미한 차이/유의한 차이)에 반영
- 자본적지출·감가상각중단이 있는 자산도 적용내용연수(개월)/당기해당월수/연수합계법 연차(k)까지 수식으로 연결(현재는 이 경우만 계산값을 그대로 둠)
